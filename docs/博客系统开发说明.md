# 博客系统开发说明

基于 Next.js 15（App Router）+ HeroUI + MySQL + Drizzle ORM 的多语言博客。**更完整的文档导航见 [文档索引.md](./文档索引.md)。**

## 技术栈

- **框架**：Next.js 15、React 19、TypeScript（`strict`）
- **UI**：HeroUI v2、Tailwind CSS 4、Framer Motion、`next-themes`
- **数据**：MySQL 8+、Drizzle ORM
- **认证**：JWT、`bcryptjs`（**未使用** NextAuth；环境变量以 `env.example` 为准）
- **包管理**：pnpm 9+（见 `package.json` 的 `packageManager`）

## 功能概览（与当前代码一致）

| 领域        | 状态说明                                                                                           |
| ----------- | -------------------------------------------------------------------------------------------------- |
| 多语言      | `middleware.ts` 重定向到 `/{zh-CN\|en-US\|ja-JP}/...`，文案见 `dictionaries/*.json`                |
| 用户认证    | `app/api/auth/*`：登录、注册、验证码、忘记/重置密码等                                              |
| 文章        | CRUD、`PostService`、Markdown/WYSIWYG 编辑（Toast UI Editor，`components/blog/simple-editor.tsx`） |
| 分类 / 标签 | 层级分类、多对多标签；管理页在 `[lang]/categories`、`[lang]/tags` 等                               |
| 个人中心    | 资料、收藏（`/api/profile/favorites`）、通知、统计等                                               |
| 评论        | **库表** `comments` 已定义，业务与 **HTTP API/前台评论区** 仍待接齐（勿与「仅有 schema」混淆）     |
| 媒体 / 设置 | **库表** `media`、`settings` 等存在；完整上传后台与设置 UI 按需迭代                                |
| 搜索        | 文章列表多为 **SQL 模糊查询（如标题等）**，非独立搜索引擎                                          |
| API 文档页  | `/{lang}/api-docs` 自动扫描 `app/api`                                                              |

## 环境与数据库

复制模板并填写真实值：

```bash
cp env.example .env.local
```

数据库连接使用 **`DB_HOST`、`DB_PORT`、`DB_USER`、`DB_PASSWORD`、`DB_NAME`**（见 `lib/db/config.ts`）。**不要**依赖文档中出现的 `DATABASE_URL` / `NEXTAUTH_*` 旧示例——以 `env.example` 为准。

### 推荐初始化方式（Drizzle）

日常开发优先用迁移或 push，与 `lib/db/schema.ts` 保持一致：

```bash
pnpm install
pnpm db:generate    # 修改 schema 后
pnpm db:migrate     # 或开发中 pnpm db:push
pnpm db:seed        # 可选：测试数据
```

`scripts/init-db.sql` 为备选方案，说明见 [数据库初始化脚本说明.md](../scripts/数据库初始化脚本说明.md)；若与 Drizzle 迁移混用，注意避免两套建表逻辑冲突。

## API 响应约定

前后端普遍使用形如下的 JSON 结构（具体类型见 `types/blog` 等）：

```typescript
interface ApiResponse<T> {
  success: boolean;
  message: string;
  data?: T;
  error?: string;
  timestamp: string;
}
```

文章列表示例（节选）：

```http
GET /api/posts?page=1&limit=10&status=published&search=关键词
```

详细 Hook 与客户端用法见 [API使用指南.md](./API使用指南.md)。

## 仓库结构（精简）

```
app/
  [lang]/           # 多语言页面
  api/              # Route Handlers（无前缀）
components/       # blog / layout / profile / ui 等
lib/
  db/               # config、schema
  services/         # PostService 等
  hooks/            # usePosts、useTags 等
dictionaries/       # 界面文案 JSON
types/
scripts/            # 备份、种子、重置、测试脚本
```

## 安全与运维提示

- 生产环境务必使用 HTTPS，妥善保管 `JWT_SECRET` 与数据库密码
- `env.example` 中的 `REDIS_URL`、`GOOGLE_*` 等为**预留/可选**配置，业务代码未强制依赖 Redis；接入前以全局检索为准
- 重置、删数等操作前使用 `pnpm db:backup`
