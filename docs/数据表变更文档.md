# 数据表变更记录

本文件专门记录 **MySQL / Drizzle 表结构** 的变更（以 **`lib/db/schema.ts`** 与 **`drizzle/`** 迁移为准）。**新记录一律写在最上方**（时间倒序）。每次变更应同步更新 **`docs/数据库表记录文档.md`**。

---

## 2026-04-21 — 补录：邮箱订阅表与验证码类型扩展

**类型**：schema（db）

### 变更说明

- 新增 `email_subscriptions` 表，用于承载访客/登录态邮件订阅与退订状态：
  - `email` 全局唯一；
  - `is_active` 表示是否有效订阅；
  - `subscribed_at` / `unsubscribed_at` 记录订阅生命周期。
- `email_verifications.type` 增加订阅相关枚举值：
  - `subscription`
  - `subscription_unsubscribe`
- 该结构用于支撑访客验证码订阅流程与登录态订阅接口统一落库。

### 涉及文件

- `lib/db/schema.ts`
- `docs/数据库表记录文档.md`
- `docs/邮件订阅与发布通知功能说明.md`

### 迁移与兼容

- 已部署环境需执行 **`pnpm db:migrate`**（或开发中 **`pnpm db:push`**）以确保接口可用。
- 若库中缺少 `email_subscriptions` 或枚举值未扩展，订阅相关接口会返回 `DB_SCHEMA_OUTDATED`。

## 2026-04-21 — categories/tags 唯一约束改为 owner 维度

**类型**：schema（db）

### 变更说明

- `categories`：移除 `name`、`slug` 的全局唯一约束，改为复合唯一索引：
  - `owner_name_unique_idx`（`owner_id`, `name`）
  - `owner_slug_unique_idx`（`owner_id`, `slug`）
- `tags`：移除 `name`、`slug` 的全局唯一约束，改为复合唯一索引：
  - `owner_tag_name_unique_idx`（`owner_id`, `name`）
  - `owner_tag_slug_unique_idx`（`owner_id`, `slug`）
- 目标是消除多用户场景下的误冲突，保持与接口层“按 owner 隔离 + owner 内唯一”的校验语义一致。

### 涉及文件

- `lib/db/schema.ts`
- `drizzle/0008_bright_lyja.sql`
- `docs/数据库表记录文档.md`

### 迁移与兼容

- 对已部署库执行 **`pnpm db:migrate`**（或开发环境 **`pnpm db:push`**）应用迁移。
- 若历史数据中同一 `owner_id` 下已存在重复 `name` / `slug`，迁移前需先清理重复数据，否则复合唯一索引创建会失败。

## 2026-04-18 — user_activities：action 加长、user_id 可空、复合索引

**类型**：schema（db）

### 变更说明

- `user_activities.action`：`varchar(50)` → `varchar(100)`，容纳统一活动打点枚举名。
- `user_activities.user_id`：**允许 NULL**（访客订阅等仅 metadata 归因、不进入个人活动流筛选的场景）。
- 新增复合索引 **`user_created_idx`**（`user_id`, `created_at`），优化「按用户拉时间线」查询。

### 涉及文件

- `lib/db/schema.ts`
- `types/blog.ts`（`UserActivity.userId` 与可空列一致）
- `app/api/profile/activities/route.ts`（序列化 `userId` 空值）
- `docs/数据库表记录文档.md`

### 迁移与兼容

- 迁移文件：`drizzle/0007_bitter_hydra.sql`（`user_id` 可空、`action` 扩长、新增 `user_created_idx`）。
- 对已部署库执行 **`pnpm db:migrate`**（或开发环境 **`pnpm db:push`**）应用上述 SQL。
- 现有行 `user_id` 均为非空，放宽为可空不影响旧数据。

## 2026-04-17 — categories/tags 增加 owner_id 并补充索引

**类型**：schema（db）

### 变更说明

- `categories` 新增 `owner_id`（非空），并新增索引 `owner_idx`。
- `tags` 新增 `owner_id`（非空），并新增索引 `owner_idx`。
- 该变更用于支撑分类/标签接口按用户隔离读写范围。

### 涉及文件

- `lib/db/schema.ts`
- `scripts/seed.ts`
- `docs/数据库表记录文档.md`

### 迁移与兼容

- 对已有数据需要补齐 `owner_id`（建议在迁移脚本中按既定策略回填），再启用非空约束。
- 回填完成后，客户端请求应通过登录态访问分类/标签接口。
