> 文档状态：**临时方案（待实现后转正式）**

> 适用阶段：当前为产品与前端开发对齐稿；待功能开发完成后，按实际实现拆分并沉淀为「产品设计文档」与「功能开发文档」。

## 1. 背景与目标

当前项目的个人中心已具备基础能力：

- 已有概览页与统计卡片（包含粉丝数、关注数、未读通知）
- 已有通知 API 与通知中心基础界面
- 已有多语言与统一的 Profile UI 体系

当前待完善重点：

- 将「粉丝数 / 关注数 / 未读通知」从静态数字升级为可行动入口
- 完成粉丝与关注列表页（关系管理）
- 升级通知中心为可执行的任务流（不仅是消息展示）

核心产品目标：

- 让个人中心从“展示页”升级为“运营与互动工作台”
- 提升通知处理率、互动回访率与关注关系转化

## 3. 可落地信息架构（IA）

## 3.1 个人中心总入口关系

在 `/{lang}/profile` 建立以下跳转闭环：

- 粉丝数卡片 -> `/{lang}/profile/followers`
- 关注数卡片 -> `/{lang}/profile/following`
- 未读通知卡片 -> `/{lang}/profile/notifications?status=unread`

形成“统计看板 -> 明细处理 -> 数据回写”的闭环。

## 3.2 Followers（粉丝）页面

目标：管理“谁在关注我”，提升互关与互动质量。

页面建议路径：

- `/{lang}/profile/followers`

页面结构：

- 顶部区：标题、副标题、总数/新增指标
- 筛选区：关系状态（互关/未互关）、时间排序、搜索
- 列表区：头像、昵称、简介、关注时间、关系标签、操作按钮
- 空状态：引导去发布内容/完善资料

主要操作：

- 回关
- 查看主页
- 可选：移除粉丝（视产品策略）

## 3.3 Following（关注）页面

目标：管理“我关注了谁”，优化关注质量与消费体验。

页面建议路径：

- `/{lang}/profile/following`

页面结构：

- 顶部区：标题、副标题
- 筛选区：排序（最近关注/最早关注/最近活跃）、关系状态
- 列表区：头像、昵称、最近活跃、关注时间、操作
- 空状态：引导去发现创作者

主要操作：

- 取消关注
- 查看主页
- 后续可扩展：特别关注

## 3.4 Notifications（通知中心）页面升级

目标：将消息列表升级为“可执行任务流”。

页面路径（沿用）：

- `/{lang}/profile/notifications`

页面结构建议：

- 概览区：未读总数、本周新增、快捷操作（全部已读/清理已读）
- 筛选区：类型、状态、时间（后续）
- 通知流：每条通知显示主动作 CTA + 次动作

通知主动作建议：

- 评论：查看评论上下文
- 点赞：查看被点赞内容
- 关注：回关 / 查看对方主页
- 提及：查看提及位置
- 系统：查看详情

## 4. 交互与状态设计原则

- 统一四态：`loading` / `empty` / `error` / `normal`
- 操作反馈统一：成功 toast + 局部更新；失败 toast + 回滚
- 未读策略：不自动全标已读，以“用户确认行为”为准
- 视觉风格：沿用 `PROFILE_GLASS_CARD` 与现有 HeroUI 体系
- i18n：三语同步（`zh-CN`、`en-US`、`ja-JP`）
