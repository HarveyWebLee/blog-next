# 个人中心社交与通知模块功能文档

> 文档状态：**正式功能文档（基于当前代码实现）**
>
> 适用范围：个人中心中与「粉丝、关注、通知」相关的前后端能力与页面交互。
>
> 更新时间：2026-07-12

## 1. 目标与范围

本模块用于将个人中心从“数据展示”升级为“可操作入口”，覆盖以下能力：

- 统计卡片快捷跳转（粉丝 / 关注 / 未读通知）
- 粉丝与关注关系页（列表、筛选、分页、操作）
- 通知中心任务化处理（筛选、标记、删除、清理已读、加载更多、主动作 CTA、回关）

不包含：

- 通知相关文案统一收敛到 `dictionaries/*.json`（当前多为组件内三语对象，见 [项目TODO.md](./项目TODO.md)）

## 3. 前端实现清单

### 3.1 统计卡片入口（T1）

- `components/profile/profile-stats.tsx`
  - `totalFollowers` -> `/{lang}/profile/followers`
  - `totalFollowing` -> `/{lang}/profile/following`
  - `unreadNotifications` -> `/{lang}/profile/notifications?status=unread`
  - 已补充 hover / focus 样式与可访问性 `aria-label`

### 3.2 粉丝页（T2 + T5）

- 路由：`app/[lang]/profile/followers/page.tsx`
- 组件：`components/profile/profile-followers.tsx`
- 关键能力：
  - 搜索、仅互关筛选、分页
  - 回关操作（调用真实接口）
  - 查看主页跳转
  - 接口失败 fallback 与提示去重
  - loading / empty / normal / fallback 四态

### 3.3 关注页（T3 + T5）

- 路由：`app/[lang]/profile/following/page.tsx`
- 组件：`components/profile/profile-following.tsx`
- 关键能力：
  - 搜索、仅互关筛选、分页
  - 取消关注（调用真实接口）
  - 查看主页跳转
  - 接口失败 fallback 与提示去重
  - loading / empty / normal / fallback 四态

### 3.4 通知页（T4 + T6）

- 路由：`app/[lang]/profile/notifications/page.tsx`
- 组件：`components/profile/profile-notifications.tsx`
- 数据接口：**主路径** `GET/PUT/DELETE /api/notifications`（非 `/api/profile/notifications`）
- 关键能力：
  - 支持 URL 初始筛选：`status=unread|read`
  - 类型筛选、已读状态筛选
  - 单条已读、批量全部已读、**清理已读**（`DELETE` + `{ clearRead: true }`）
  - **加载更多**（`page` 递增、`limit=20`、`hasNextPage`）
  - 单条删除
  - 主动作 CTA（按通知类型）
  - follow 类型支持“回关”直连（调用关注接口）

## 4. 后端接口实现清单

### 4.1 关系接口（T5）

- `GET /api/profile/followers`
  - 文件：`app/api/profile/followers/route.ts`
  - 支持：`page`、`limit`、`search`、`mutualOnly`
- `GET /api/profile/following`
  - 文件：`app/api/profile/following/route.ts`
  - 支持：`page`、`limit`、`search`、`mutualOnly`
- `POST /api/profile/follow`
  - 文件：`app/api/profile/follow/route.ts`
  - 关注用户，含重复关注/自关注校验，并尝试写入 follow 通知
- `DELETE /api/profile/follow/[targetUserId]`
  - 文件：`app/api/profile/follow/[targetUserId]/route.ts`
  - 取消关注

### 4.2 通知接口（前台主路径）

- `GET /api/notifications`：列表（`page`、`limit`、`type`、`isRead` 等筛选）
- `PUT /api/notifications`：批量已读（`markAllAsRead` 或 `notificationIds`）
- `DELETE /api/notifications`：批量删除；`clearRead=true` 清理全部已读
- `PUT /api/notifications/[id]`：单条已读等更新
- `DELETE /api/notifications/[id]`：单条删除

> `/api/profile/notifications` 仍存在，与上组接口并行；新功能应优先走 `/api/notifications`（见 [项目TODO.md](./项目TODO.md) 收敛项）。

### 评论通知写入（2026-07-12）

评论闭环已接入站内通知创建（服务层 `lib/services/notification.service.ts`）：

| 事件                  | 接收人                      | 类型      |
| --------------------- | --------------------------- | --------- |
| 根评论提交（非 spam） | 文章作者                    | `comment` |
| 回复提交（非 spam）   | 被回复者 + 文章作者（去重） | `comment` |
| 审核通过              | 评论作者                    | `comment` |
| 审核拒绝（spam）      | 评论作者                    | `system`  |

`data` 含 `postSlug` / `postId` / `commentId` / `kind`，与通知中心主动作跳转一致。匿名评论作者无 `authorId` 时不发审核结果通知。
