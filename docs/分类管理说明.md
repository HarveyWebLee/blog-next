# 分类管理说明

> 本文档由原《分类管理说明》与《分类实现摘要》合并而成：**前半**为功能与使用方式，**后半**为实现结构与 API/校验细节，避免两篇文档重复维护。

前台与管理页路径均带语言前缀，形如 `/{zh-CN|en-US|ja-JP}/categories/...`；下文中的 `/categories` 均指该前缀下的路由。

---

## 一、功能概述

分类管理提供完整的增删改查能力：

- **分类列表**：搜索、筛选、分页
- **创建 / 编辑**：支持层级（父分类）、排序、激活状态
- **删除**：有子分类或关联文章时禁止删除
- **状态**：可快速切换激活 / 停用

---

## 二、使用说明

### 2.1 进入管理

在分类列表页（`/categories`）中，具备管理权限的用户可通过「分类管理」入口进入管理界面（具体以当前 `app/[lang]/categories/page.tsx` 中的权限判断为准）。

### 2.2 创建分类

1. 在管理页点击「创建分类」
2. 填写名称（必填）、标识 slug（前端默认预填随机值，提交时为必填）、描述、父分类、排序、状态
3. 保存

### 2.3 编辑与删除

在列表操作菜单中选择编辑或删除；删除前若仍存在子分类或关联文章，接口会拒绝并提示。

### 2.4 状态管理

列表中可通过开关切换激活 / 停用。

## 三、API 参考

基础路径：`/api/categories`（不受语言前缀影响）。

### 3.1 列表

`GET /api/categories`

**查询参数**：`page`、`limit`、`search`、`isActive`、`parentId`、`sortBy`、`sortOrder`（`asc` / `desc`）。

**响应示例**（结构以实际接口为准）：

```json
{
  "data": {
    "data": [
      {
        "id": 1,
        "name": "技术分享",
        "slug": "tech",
        "description": "技术相关的文章分享",
        "parentId": null,
        "sortOrder": 1,
        "isActive": true,
        "createdAt": "2025-09-21T05:51:31.000Z",
        "updatedAt": "2025-09-21T05:51:31.000Z"
      }
    ],
    "pagination": {
      "page": 1,
      "limit": 10,
      "total": 3,
      "totalPages": 1,
      "hasNext": false,
      "hasPrev": false
    }
  },
  "message": "分类列表获取成功",
  "success": true,
  "timestamp": "2025-09-26T12:31:11.417Z"
}
```

### 3.2 创建

`POST /api/categories`

**请求体示例**：

```json
{
  "description": "分类描述",
  "isActive": true,
  "name": "分类名称",
  "parentId": 1,
  "slug": "category-slug",
  "sortOrder": 0
}
```

服务端会校验 `name` 与 `slug` 必填，并在当前登录用户 `ownerId` 范围内检查名称与 slug 的唯一性。

### 3.3 单条查询 / 更新 / 删除

- `GET /api/categories/[id]`：单条详情（当前实现会返回 `postCount`）
- `PUT /api/categories/[id]`：部分更新，同样校验唯一性
- `DELETE /api/categories/[id]`：删除前校验子分类与文章引用

## 四、实现结构

### 4.1 目录与主要文件

```
app/
├── api/categories/
│   ├── route.ts                 # 列表 GET、创建 POST
│   └── [id]/route.ts            # 单条 GET / PUT / DELETE
├── [lang]/categories/
│   ├── page.tsx                 # 分类展示 + 管理入口
│   └── manage/
│       ├── layout.tsx
│       ├── page.tsx             # 管理列表
│       ├── create/page.tsx      # 新建
│       └── edit/[id]/page.tsx   # 编辑
types/
└── blog.ts                      # Category、CategoryQueryParams 等
```

### 4.2 API 层要点

- 列表：分页、搜索、按 `isActive` / `parentId` 筛选、排序
- 创建：层级 `parentId`、排序字段、唯一性校验
- 删除：避免误删仍有子节点或文章的分类

### 4.3 管理界面要点

- 列表：搜索筛选、分页（含“加载更多”）、删除确认、状态切换
- 创建 / 编辑：表单校验、slug 生成或编辑、父分类选择（编辑时会排除自身及后代防止循环引用）、加载与错误提示
- 布局：`manage/layout.tsx` 统一子页面布局

### 4.4 类型定义

见 `types/blog.ts`：`Category`、`CategoryQueryParams` 等与分类相关的类型应与 `lib/db/schema` 及 API 返回保持一致。

## 五、数据表与校验

### 5.1 表字段（概念）

`categories` 表主要字段包括：`id`、`name`、`slug`、`description`、`parentId`、`sortOrder`、`isActive`、`createdAt`、`updatedAt`。**权威结构以 Drizzle schema（`lib/db/schema.ts`）与迁移为准。**

### 5.2 校验与安全

- 服务端校验输入，保证类型与业务规则（唯一性、层级合法、删除条件）
- 分类读写接口当前均要求 `Authorization: Bearer`，并按 `ownerId` 做数据隔离
- 客户端可做基础校验与防抖搜索，以减轻请求压力

## 六、层级、体验与扩展方向

### 6.1 层级与搜索

- 支持多级分类；避免父链成环
- 可按名称、描述、slug 搜索，并按状态筛选

### 6.2 体验

- 响应式布局；列表加载态与操作反馈（错误提示、确认框）

### 6.3 后续可扩展能力（按需）

- 分类图标 / 颜色主题、批量操作、导入导出、使用统计、缓存或搜索索引优化等

## 七、权限说明

若当前代码中对「进入管理」仍为非严格 RBAC，部署前应：

1. 与全局认证（如 JWT / Session）对齐
2. 仅允许管理员或内容角色访问 `manage` 路由与写操作 API

具体以 `app/api/categories` 与 `app/[lang]/categories` 中的实际校验为准。
