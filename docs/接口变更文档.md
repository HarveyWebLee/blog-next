# 接口变更记录

本文件专门记录 **HTTP API（`app/api`）** 的变更历史，便于对接与回归。**新记录一律写在最上方**（时间倒序）。每次变更应同步更新 **`docs/接口文档.md`** 中的对应条目。

---

## 2026-04-23 — API 统一错误拦截、日志规范化与路由包装收口

**类型**：refactor（api / infra / logging）

### 变更说明

- `app/api/**/route.ts` 全量收口到 `defineApiHandlers`：未捕获异常统一由包装器处理，减少每个接口重复 `catch + logger.exception + 500 return` 模板代码。
- `defineApiHandlers` 增加默认 JSON 500 兜底（`success=false`、`message`、`timestamp`），避免未映射异常回落为 HTML 错误响应。
- 新增并接入统一服务端日志能力：
  - `lib/server/logger.ts`：分级日志（debug/info/warn/error）与统一输出格式；
  - `lib/server/log-redaction.ts`：日志内容与 meta 脱敏；
  - `lib/server/route-alert.ts`：未捕获路由异常可选 webhook 告警桥接。
- 新增 `scripts/check-api-handler-wrapping.ts`，并接入 `pnpm precommit`，用于校验所有 `route.ts` 是否使用包装器导出。

### 涉及文件

- `lib/server/define-api-handlers.ts`
- `lib/server/logger.ts`
- `lib/server/log-redaction.ts`
- `lib/server/route-alert.ts`
- `scripts/check-api-handler-wrapping.ts`
- `package.json`
- `env.example`
- `app/api/**/route.ts`（全量 Route Handlers）
- `docs/接口文档.md`

### 对客户端影响

- 成功响应与业务 4xx 分支保持原有语义；
- 未捕获 500 错误响应改为统一 JSON 结构，返回内容更稳定，且不再泄露 SQL/堆栈细节；
- 建议客户端统一基于 `success`/`message`（及业务侧 `code`）处理失败提示，避免依赖底层错误文本。

## 2026-04-23 — 密码字段新增应用层传输封装（passwordTransport）

**类型**：feat（api / auth / posts / security）

### 变更说明

- 新增 `GET /api/auth/password-transport-params`：下发密码传输参数（RSA 公钥、`keyId`、时钟偏差窗口）。

- 服务端新增 nonce + timestamp + sign 校验与防重放（Redis 优先、内存兜底）。

## 2026-04-23 — 订阅接口数据库错误响应与错误码规范化

**类型**：fix（api / subscriptions）

### 变更说明

- `GET|POST|DELETE /api/subscriptions`：数据库**连接失败**（如 ETIMEDOUT）统一返回 **503**，`code=DB_UNAVAILABLE`，对用户文案与「缺表」区分。
- 移除响应体中的原始 `error` 字符串（避免泄露 Failed query/SQL）；开发环境 **500** 时可含 `debug` 摘要。
- 与首页、文章列表等服务端日志对齐：使用结构化 `[db-error]` 单行 JSON，替代默认 `console.error` 刷屏堆栈与整段 SQL。

### 涉及文件

- `app/api/subscriptions/route.ts`
- `lib/utils/mysql-error.ts`
- `lib/services/post.service.ts`
- `app/[lang]/page.tsx`
- `lib/utils/api-docs-endpoint-meta.ts`
- `docs/接口文档.md`

### 对客户端影响

- 依赖 `error` 字段展示订阅失败原因的前端需改为依据 `code`（`DB_UNAVAILABLE` / `DB_SCHEMA_OUTDATED` / `INTERNAL_ERROR`）或固定 `message`。

## 2026-04-22 — API 文档高级能力：OpenAPI 导出、版本统计与文档调用统计

**类型**：feat（api / api-docs）

### 变更说明

- 增强 `GET /api/api-docs`：
  - 新增 `format=openapi`，直接返回 OpenAPI 3.0.3 JSON；
  - 新增 `download=true`，在导出时附带下载响应头；

- 文档页能力增强：
  - 在 `/{lang}/api-docs` 增加“导出 OpenAPI”按钮；
  - 展示版本标注覆盖统计与文档调用统计；
  - 新增“版本筛选”下拉，支持按具体版本或“未标注版本”查看与导出。

## 2026-04-22 — 通知接口安全模型落地（本人隔离 + 超级管理员跨用户）

**类型**：feat（api / notifications）

### 变更说明

- `api/notifications*` 从示例返回改为真实数据库实现（`user_notifications`）：
  - `GET /api/notifications`：返回分页通知列表，支持 `type`、`isRead` 过滤；
  - `POST /api/notifications`：创建通知；
  - `GET /api/notifications/{id}`：获取单条通知；
  - `PUT /api/notifications/{id}`：更新通知（支持 `isRead`、`title`、`content`、`type`、`data`）；
  - `DELETE /api/notifications/{id}`：删除通知。
- 鉴权与权限模型统一：
  - 全部接口要求 `Authorization: Bearer`；
  - 普通用户仅可读写自己的通知（`notification.userId === currentUser.userId`）；
  - 超级管理员 root 会话可跨用户管理通知。
- `GET /api/notifications` 支持超级管理员通过 `query.userId` 指定目标用户范围查询。

### 涉及文件

- `app/api/notifications/route.ts`
- `app/api/notifications/[id]/route.ts`
- `lib/utils/api-docs-endpoint-meta.ts`
- `docs/接口文档.md`

### 对客户端影响

- 未携带 Bearer 调用 `api/notifications*` 将返回 401。
- 普通用户跨用户访问通知将返回 403。
- 超级管理员可在管理场景下通过 `userId` 查询与维护其他用户通知。

## 2026-04-22 — 安全加固：高危接口鉴权、互动防刷与可见性收敛

**类型**：fix（api / security）

### 变更说明

- 高危运维接口统一改为仅超级管理员可访问（`requireInMemorySuperRoot`）：`GET /api/seed`、`GET /api/test-db`、`GET /api/test-env`、`GET /api/proxy/{...path}`。
- 对匿名可调用的互动端点增加防刷限流（429 + `Retry-After`）：
  - `POST /api/comments`（按用户或 IP + 文章限流）
  - `POST /api/posts/{id}/share`（按用户或 IP + 文章限流）
