# 接口变更记录

本文件专门记录 **HTTP API（`app/api`）** 的变更历史，便于对接与回归。**新记录一律写在最上方**（时间倒序）。每次变更应同步更新 **`docs/接口文档.md`** 中的对应条目。

---

## 2026-04-17 — PATCH 管理用户：禁止修改根账户角色/状态

**类型**：feat（安全约束 + 前台）

### 变更说明

- **`PATCH /api/admin/users/{id}`**：当路径 `id` 与当前超级管理员令牌中的 `userId`（根账户）相同时，返回 **403**，不修改数据库。
- **账户管理弹窗**：目标用户为根账户（详情中 `role` 为展示用的 `super_admin`）时，不再显示「角色」「状态」下拉，仅只读展示；「保存」禁用。
- 修正打开详情时曾将 `super_admin` 误映射为 `admin` 的状态，避免误发起 PATCH。

### 涉及文件

- `app/api/admin/users/[id]/route.ts`
- `components/profile/profile-accounts-admin.tsx`
- `lib/utils/api-docs-endpoint-meta.ts`、`docs/接口文档.md`

### 对调用方影响

- 自动化脚本若对根账户本人发 PATCH，将收到 403；正常仅改其他用户时不影响。

## 2026-04-17 — 超级管理员与互动接口去兼容化清理

**类型**：refactor（auth / api）

### 变更说明

- 超级管理员统一为真实 DB 用户模型，不再依赖 `userId=0` 约定。
- `getAuthUserFromRequest` 改为仅接受正整数 `userId`，删除旧 token 特判。
- 点赞/收藏/engagement 删除 `userId=0` 限制分支，登录用户（含 DB 超管）均按同一逻辑处理。
- 注册、资料邮箱冲突校验、关于页站长资料统一按真实超级管理员 `user_id` 查询。

### 涉及文件

- `lib/config/super-admin.ts`、`lib/utils/request-auth.ts`
- `app/api/posts/engagement/route.ts`、`app/api/posts/[id]/like/route.ts`、`app/api/posts/[id]/favorite/route.ts`
- `app/api/auth/login/route.ts`、`app/api/auth/register/route.ts`、`lib/services/profile-email.service.ts`、`lib/services/about-owner.service.ts`

### 对调用方影响

- 不再支持历史 `userId=0` 会话；需重新登录获取新版 token。
- 超级管理员与普通登录用户在互动接口上行为一致。

## 2026-04-17 — 超级管理员改为真实 DB 用户

**类型**：fix（auth / api）

### 变更说明

- `resolveSuperAdminLogin` 改为登录成功后自动确保 `users` 超管行存在（`role=admin`），并签发 `role=super_admin + isRoot=true` 但 `userId` 使用真实 DB id。
- `refresh` 支持旧 token平滑刷新为新 token（真实 DB `userId`）。
- 超管 profile 自动迁移：若仅存在 `user_profiles.user_id=<super_admin_id>`，会在首次登录时复制到真实超管 `user_id`。
- 点赞/收藏/engagement 的 限制仅对 `userId=0` 生效；新版 DB 超管可正常使用互动功能。

### 涉及文件

- `lib/config/super-admin.ts`、`app/api/auth/login/route.ts`、`app/api/auth/refresh/route.ts`
- `app/api/posts/engagement/route.ts`、`app/api/posts/[id]/like/route.ts`、`app/api/posts/[id]/favorite/route.ts`
- `app/api/auth/register/route.ts`、`lib/services/profile-email.service.ts`、`lib/services/about-owner.service.ts`

### 对调用方影响

- 超管登录后获得真实 `userId`，可参与点赞/收藏；旧 refresh token 可无感升级。
- 若系统仍存在 `userId=0` 超管 token，其互动接口限制保持不变（POST 点赞/收藏 403、engagement 占位）。

## 2026-04-17 — 文章互动接口：JWT 归一化、超级管理员约束与点赞落库健壮性

**类型**：fix（行为细化 + 排错增强）

### 变更说明

- **`getAuthUserFromRequest`**：`userId` 支持数字字符串归一化；拒绝非法的 `userId=0`（仅允许与刷新令牌一致的超级管理员：`role=super_admin` 且 `isRoot=true`）。
- **`GET /api/posts/engagement`**：超级管理员直接返回未赞未收藏占位，避免对 `user_post_likes` / `user_favorites` 使用无意义 `user_id=0`；查询结果 `postId` 统一 `Number` 以兼容驱动 `bigint`；失败时 `console.error` 附带 MySQL 驱动错误解析。
- **`POST /api/posts/{id}/like`**：超级管理员 **403**；`user_post_likes` 插入显式 `created_at`；`like_count` 改为读改写并经 `toJsonSafeInt` 输出，避免 `BigInt` 序列化异常及表达式默认值差异。
- **`POST /api/posts/{id}/favorite`**：超级管理员 **403**；插入收藏显式 `created_at`；失败日志增强。

### 涉及文件

- `lib/utils/request-auth.ts`、`lib/utils/tool.ts`（`toJsonSafeInt`）
- `app/api/posts/engagement/route.ts`、`app/api/posts/[id]/like/route.ts`、`app/api/posts/[id]/favorite/route.ts`
- `lib/utils/api-docs-endpoint-meta.ts`、`docs/接口文档.md`

### 对调用方影响

- 使用 **超级管理员**（JWT `userId=0` + `isRoot`）时：批量互动接口仍为 **200**（占位）；**点赞/收藏 POST 为 403**，需换用数据库用户登录后再操作。
- 普通用户契约不变；若此前因 `created_at` / 计数序列化导致 500，应随本次修复消除。
