# 分类页动态阴影效果说明（与当前实现对齐）

本文档聚焦 `/{lang}/categories` 页面的阴影与光效实现，按当前代码状态区分为「已启用」与「可扩展（未默认启用）」两部分。

## 当前已启用

### 1) CSS 主导的阴影层次

- 文件：`app/[lang]/categories/categories.scss`
- 作用对象：`.category-card-modern`、`.search-content`、`.stat-card-modern`
- 现状：
  - 卡片默认阴影 + hover 阴影 + 顶部渐变条展开；
  - 分类层级（`level-1/2/3`）使用不同色系阴影；
  - 暗色主题使用 `html.dark` 分支，不依赖 `prefers-color-scheme`；
  - 移动端（`max-width: 768px`）自动降低阴影强度。

### 2) JS 阴影增强（已在分类卡片挂载）

- 文件：`app/[lang]/categories/shadow-effects.ts`
- 页面接入：`app/[lang]/categories/page.tsx` 中 `CategoryCard` 的 `useEffect`
- 当前实际调用：
  - `applyLevelShadow(...)`
  - `applyHoverShadow(...)`
  - `applyClickShadow(...)`
- 说明：JS 事件用于补充交互细节，主体视觉仍由 SCSS 控制。

### 3) 减少动画偏好支持

- 文件：`app/[lang]/categories/categories.scss`
- 通过 `@media (prefers-reduced-motion: reduce)` 关闭主要动画/过渡，避免高动态效果影响可访问性。

## 可扩展能力（当前未默认启用）

`shadow-effects.ts` 还提供了以下方法，但分类页默认流程未主动启用：

- `applyFocusShadow`
- `applyPulseShadow`
- `applyFloatShadow`
- `applyGlowShadow`
- `applyThemeShadow`

这些方法可用于后续交互实验，但若启用建议先评估滚动条抖动、暗色对比度与性能影响。

## 实现边界与注意点

- 阴影是「视觉增强」而非状态语义，不应替代可访问性状态提示（如 `aria-*`、文本反馈）。
- 当前样式里存在较多演进阶段保留规则；以页面实际挂载类名与调用逻辑为准，不建议仅凭某个类名判断功能已启用。
- 若后续调整交互，优先在 `categories.scss` 与 `CategoryCard` 一并改动，避免 CSS 与 JS 阴影逻辑脱节。
