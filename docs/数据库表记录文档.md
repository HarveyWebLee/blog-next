# 数据库表记录（全量）

本文档描述 **`lib/db/schema.ts`** 中 Drizzle 定义的 MySQL 表，与 **`drizzle/`** 已应用迁移保持一致。表结构变更时须同步更新本文件，并在 **`docs/数据表变更文档.md`** 顶部追加记录。权威源码：**`lib/db/schema.ts`**。

---

## 总览

| Drizzle 导出名       | MySQL 表名            | 用途摘要                                   |
| -------------------- | --------------------- | ------------------------------------------ |
| `users`              | `users`               | 账号：登录名、邮箱、密码哈希、角色、状态等 |
| `userProfiles`       | `user_profiles`       | 用户扩展资料（与 `users` 一对一）          |
| `userPreferences`    | `user_preferences`    | 用户键值偏好设置                           |
| `userActivities`     | `user_activities`     | 用户活动日志                               |
| `userFavorites`      | `user_favorites`      | 用户收藏文章                               |
| `userPostLikes`      | `user_post_likes`     | 用户对文章的点赞（复合主键 userId+postId） |
| `userFollows`        | `user_follows`        | 用户关注关系                               |
| `userNotifications`  | `user_notifications`  | 用户通知                                   |
| `categories`         | `categories`          | 文章分类（支持 parent 层级）               |
| `tags`               | `tags`                | 标签                                       |
| `posts`              | `posts`               | 文章主表（含浏览量、点赞数、可见性等）     |

| `comments`           | `comments`            | 评论（含嵌套 parent）                      |
| `media`              | `media`               | 媒体元数据                                 |
| `emailVerifications` | `email_verifications` | 邮箱验证码（多业务类型枚举）               |

| `settings`           | `settings`            | 系统键值配置                               |

## 关键字段与约束（摘要）

### users

- **主键**：`id`（自增）
- **唯一**：`username`、`email`
- **枚举**：`role` — `admin` \| `author` \| `user`；`status` — `active` \| `inactive` \| `banned`

### posts

- **主键**：`id`
- **唯一**：`slug`
- **外键语义**：`authorId` → `users.id`；`categoryId` → `categories.id`（可空）
- **枚举**：`status` — `draft` \| `published` \| `archived`；`visibility` — `public` \| `private` \| `password`
- **计数缓存**：`viewCount`、`likeCount`（与点赞表逻辑配合）

### user_post_likes

- **主键**：`(user_id, post_id)` 复合主键，保证同一用户对同一文章仅一条点赞记录

### email_verifications.type

- **枚举**：含 `register`、`reset_password`、`change_email`、`subscription`、`subscription_unsubscribe` 等（见 schema）

### email_subscriptions

- **唯一**：`email`（全局）
