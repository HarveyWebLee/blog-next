# 邮件订阅与发布通知功能说明

## 1. 背景与目标

本功能用于实现博客文章发布后的邮件通知能力，满足以下业务要求：

- 未登录用户可输入邮箱进行订阅。
- 已登录用户无需输入邮箱，直接使用注册邮箱进行订阅/取消订阅。
- 同一邮箱不允许重复订阅。
- 用户默认不订阅，订阅后按钮切换为“取消订阅”。
- 当文章首次发布时，向所有有效订阅邮箱发送文章简介与文章链接。

## 2. 实现范围总览

本次实现包含以下模块：

- 数据层：新增订阅表 `email_subscriptions`。
- API 层：新增 `/api/subscriptions`（查询状态/订阅/取消订阅）。
- 前端层：改造 `BlogSidebar` 订阅卡片，支持登录态与按钮状态切换。
- 通知层：文章发布时触发订阅邮件发送。
- 验证层：完成匿名订阅、登录订阅、取消订阅、发布通知、草稿转发布通知回归测试。

## 3. 数据库设计

### 3.1 新增表

文件：`lib/db/schema.ts`

新增表：`email_subscriptions`

关键字段：

- `email`：订阅邮箱，`unique`，防止重复订阅。
- `userId`：可选，已登录用户订阅时关联用户 ID。
- `isActive`：是否有效订阅。
- `subscribedAt`：首次订阅时间。
- `unsubscribedAt`：取消订阅时间。

### 3.2 迁移

执行方式（已验证）：

```bash
pnpm db:generate
pnpm db:migrate
```

> 若未迁移，`/api/subscriptions` 会因表不存在而报错。

## 4. API 设计与行为

文件：`app/api/subscriptions/route.ts`

### 4.1 查询订阅状态

- `GET /api/subscriptions?email=<邮箱>`
- 返回：`{ email, isSubscribed }`

用途：

- 已登录状态下初始化按钮显示（订阅/取消订阅）。

### 4.2 订阅

- `POST /api/subscriptions`
- 入参：`{ email, userId? }`

规则：

- 邮箱格式校验，不合法返回 400。
- 若邮箱已存在且 `isActive=true`：返回成功并提示“已订阅，无需重复订阅”。
- 若邮箱已存在但已取消：重新激活该记录。
- 若邮箱不存在：新建订阅记录。

### 4.3 取消订阅

- `DELETE /api/subscriptions`
- 入参：`{ email }`

规则：

- 有效订阅记录置为 `isActive=false` 并记录 `unsubscribedAt`。
- 若本就未订阅，返回成功（幂等）。

## 5. 前端交互设计

文件：`components/blog/blog-sidebar.tsx`

### 5.1 未登录用户

- 显示邮箱输入框 + “订阅”按钮。
- 点击后调用 `POST /api/subscriptions`。
- 按钮文案保持“订阅”（不切换状态）。

### 5.2 已登录用户

- 不显示输入框。
- 默认显示“订阅”按钮（使用登录邮箱）。
- 组件初始化调用 `GET /api/subscriptions` 查询当前邮箱订阅状态：
  - 未订阅：按钮显示“订阅”
  - 已订阅：按钮显示“取消订阅”
- 点击“取消订阅”调用 `DELETE /api/subscriptions`。

### 5.3 关键点

- 登录邮箱来源：`useAuth()` 的 `user.email`。
- 提交中禁用/加载状态：避免重复提交。
- 订阅结果使用统一消息提示反馈。

## 6. 发布通知链路

### 6.1 邮件发送函数

文件：`lib/utils/email.ts`

新增：

- `sendPostPublishedEmail({ to, title, excerpt, postUrl })`

内容：

- 发送文章标题、简介和“阅读全文”链接。

### 6.2 订阅服务

文件：`lib/services/subscription.service.ts`

新增：

- `notifyOnPostPublished(post)`

逻辑：

- 查询 `isActive=true` 的订阅邮箱。
- 逐个发送邮件，统计成功/失败数量。
- 输出发送汇总日志。

### 6.3 接入发布点

文件：

- `app/api/posts/route.ts`
- `app/api/posts/[id]/route.ts`

触发时机：

- 创建文章且状态为 `published`。
- 更新文章从“非 published”变为 `published`（`PUT`/`PATCH`）。

## 7. 本次实现中的关键修复点

### 7.1 发布触发判断修复（重要）

问题：

- 文章服务返回结构存在嵌套（`posts` 子对象），直接读取 `newPost.status` 可能失效。

修复：

- 统一使用 `const postCore = (result as any)?.posts || result` 再读取 `status/title/slug/excerpt`。

效果：

- 避免“实际发布了，但未触发订阅通知”的问题。

### 7.2 可观测性增强

新增日志：

- 单封邮件发送成功日志（含收件邮箱、`messageId`）。
- 单封邮件失败日志（含收件邮箱）。
- 批量发送汇总日志（总数/成功数/失败数）。

作用：

- 便于快速定位“未收到邮件”到底是未触发、发送失败还是邮箱侧分流问题。

## 8. 联调与回归结论

已完成并通过：

- 匿名订阅流程（含重复订阅防重）。

## 9. 部署与配置说明

### 9.1 邮件配置

参考变量：

- `SMTP_HOST`
- `SMTP_PORT`
- `SMTP_SECURE`
- `SMTP_USER`
- `SMTP_PASS`

若未正确配置，邮件发送阶段会失败（例如 SMTP 鉴权报错）。

### 9.2 链接域名/IP

订阅邮件中的文章链接来源：

- `NEXT_PUBLIC_APP_URL`（未配置时回退 `http://localhost:3000`）

无域名场景建议直接配置服务器公网 IP，例如：

```env
NEXT_PUBLIC_APP_URL=http://<SERVER_IP>:13001
```
