# 数据库重置与验证指南

本文说明如何在**本地/开发环境**备份、清空并重新初始化数据库。**生产环境**请使用 `pnpm db:reset:prod` 前务必再次确认影响范围，并已有可靠备份。

库表结构**仅以 Drizzle**（`lib/db/schema.ts` + `pnpm db:migrate` / `docker compose` 的 `db-migrate`）为准；不再使用独立 SQL 初始化脚本。

## 环境变量

确保 `.env.local`（或 `.env`）中已配置（与 `env.example` 一致）：

- `DB_HOST`、`DB_PORT`、`DB_USER`、`DB_PASSWORD`、`DB_NAME`

快速检测：

```bash
pnpm test:db:connect
pnpm test:db:env
```

## 推荐流程（开发环境）

```bash
# 1. 备份（备份文件默认在 ./.backups/ 下，JSON）
pnpm db:backup

# 2. 开发环境重置：清空并按项目脚本逻辑重新初始化（见 scripts/reset-database-dev.ts）
pnpm db:reset:dev

# 3. 如需重新灌入种子数据
pnpm db:seed

# 4. 可选：全系统校验脚本
pnpm db:verify
```

**生产或谨慎场景**可使用：

```bash
pnpm db:reset:prod
```

（执行前请阅读 `scripts/reset-database-prod.ts` 中的提示与约束。）

## 分步操作（不跑一体化重置脚本时）

```bash
pnpm db:backup
pnpm db:clear          # 或视情况手动清空表
pnpm db:generate       # schema 变更后需要时执行
pnpm db:migrate        # 或开发中 pnpm db:push
pnpm db:seed
pnpm test:db:connect
```

## `db:generate` 何时需要执行

- **需要执行**：
  - 仓库首次缺少 `drizzle/` 目录；
  - `lib/db/schema.ts` 有变更，需生成新迁移。
- **不需要执行**：
  - schema 未变，仅部署/重启环境时；此时直接 `pnpm db:migrate` 或 Compose `db-migrate`。

建议：迁移文件（`drizzle/*.sql`）应与 schema 变更一起提交，避免部署时出现 `"/drizzle": not found`。

## 验证功能

### 启动应用

```bash
pnpm dev
```

### 建议手工点的页面（均需语言前缀）

- 注册/登录：`/{lang}/auth/register`、`/{lang}/auth/login`
- 文章列表与后台：`/{lang}/blog`、`/{lang}/blog/manage`
- 分类、标签：`/{lang}/categories`、`/{lang}/tags`
- 个人中心：`/{lang}/profile` 及子页面
- 接口文档：`/{lang}/api-docs`（仅超级管理员；见 `docs/接口文档系统说明.md`）

### API 脚本测试

```bash
pnpm test:api
```

> **说明**：仓库根 `package.json` 中**没有** `test:api:tags` 脚本；标签相关请通过 `test:api` 或 `/{lang}/api-docs` 验证。

## 故障排除

### 数据库连接失败

- 确认 MySQL 进程或容器已启动，端口与 `DB_PORT` 一致
- Windows 若用 Docker，注意防火墙与端口映射

### 迁移失败

```bash
# 仅在明确后果时清理迁移产物，并与团队流程一致
rm -rf drizzle/
pnpm db:generate
pnpm db:migrate
```

若使用 Compose，且报错类似：

- `COPY drizzle ./drizzle`
- `failed to calculate checksum ... "/drizzle": not found`

处理步骤：

1. 在仓库根目录执行 `pnpm db:generate`；
2. 确认已生成 `drizzle/*.sql`；
3. 重新执行 `docker compose --env-file deploy/.env.docker --profile migrate run --rm db-migrate`；
4. 将新迁移文件提交到版本库。

### 查看备份

```bash
ls -la .backups/
```

备份为 JSON，用途为灾备与迁移参考；若需「一键还原」需在团队内约定还原脚本或流程。

## 校验清单（重置后）

- [ ] 表结构与 `lib/db/schema.ts` 一致（可配合 Drizzle Studio：`pnpm db:studio`）
- [ ] 能完成注册/登录
- [ ] 文章、分类、标签核心流程可用
- [ ] `pnpm test:api` 或通过 api-docs 抽样调用无重大问题
