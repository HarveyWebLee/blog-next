# 认证实现说明（按当前代码）

## 概述

当前认证体系基于 JWT + `bcryptjs`，覆盖登录、注册、验证码、刷新令牌等能力，并支持“内存态超级管理员 root 会话”。

## 已实现能力

### 登录与刷新

- `POST /api/auth/login`：支持用户名或邮箱登录。
- `POST /api/auth/refresh`：使用 `refreshToken` 换取新 `accessToken`。
- 普通数据库用户登录会校验 `status === active`，非激活用户返回 403。

### 注册与验证码

- `POST /api/auth/register`：支持常规注册；`useEmailVerification=true` 时需校验 `register` 验证码。
- `POST /api/auth/send-verification-code`：支持五种验证码类型（`register`、`reset_password`、`change_email`、`subscription`、`subscription_unsubscribe`）。
- `POST /api/auth/verify-code`：校验并消费验证码。

### 忘记/重置密码（当前状态）

- `POST /api/auth/forgot-password`、`POST /api/auth/reset-password` **目前为占位实现**：
  - 忘记密码接口当前以日志模拟发送；
  - 重置密码接口使用进程内 `Map` 模拟 token 校验，不是数据库持久化流程。
- 生产可用的密码找回链路需要后续补齐（持久化 token、邮件模板与回调链接安全策略）。

### 超级管理员 root 会话

- 配置来源：`lib/config/super-admin.ts` + `env.example` 中 `SUPER_ADMIN_*`。
- 登录成功后会确保存在真实 `users` 超管行，并签发 `role=super_admin`、`isRoot=true` 的 JWT。
- `refresh` 对该会话同样生效，并统一回到真实 DB `userId`。

## 页面路由与布局

- 认证页面路径：`/{lang}/auth/login`、`/{lang}/auth/register`、`/{lang}/auth/forgot-password`、`/{lang}/auth/reset-password`。
- 认证页通过 `components/layout/conditional-layout.tsx` 隐藏全站 Header/Footer，使用独立全屏背景布局。

## 关键文件

- API：`app/api/auth/*/route.ts`
- 认证上下文：`lib/contexts/auth-context.tsx`
- Token 与密码工具：`lib/utils/auth.ts`
- 超级管理员配置：`lib/config/super-admin.ts`
- 认证页布局：`components/layout/conditional-layout.tsx`

## 环境变量（认证相关）

- `DB_*`
- `JWT_SECRET`、`JWT_REFRESH_SECRET`
- `JWT_EXPIRES_IN`、`JWT_REFRESH_EXPIRES_IN`
- `SUPER_ADMIN_*`（可选）
- `SMTP_*`（验证码邮件）

以 `env.example` 为准，不使用历史 `DATABASE_URL` / `NEXTAUTH_*` 口径。

## 本地运行

```bash
pnpm install
pnpm db:push
pnpm dev
```

## 风险与后续建议

- 优先补齐忘记/重置密码正式流程（当前占位实现不建议生产使用）。
- 对登录与验证码接口增加更细粒度限流与审计事件（如失败次数、IP 维度统计）。
