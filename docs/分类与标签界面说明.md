# 分类与标签界面说明

本文档描述当前分类/标签页面在重构后的实现状态（以仓库代码为准）。

## 路由与文件

```text
app/[lang]/categories/page.tsx          分类页（含页内管理入口与弹窗）
app/[lang]/tags/page.tsx                标签页（含页内管理入口与弹窗）
app/[lang]/categories/manage/...        分类独立管理路由
app/[lang]/tags/manage/...              标签独立管理路由
components/layout/user-nav.tsx          用户下拉菜单（分类管理/标签管理入口）
components/ui/tag-cloud.tsx             标签云复用组件
lib/hooks/useCategories.ts              分类数据请求与分页“加载更多”
lib/hooks/useTags.ts                    标签数据请求
```

## 本次重构后的行为

### 1) 页面侧（`/{lang}/categories` 与 `/{lang}/tags`）

- 分类页与标签页都支持搜索、筛选、分页（分类含“加载更多”）、统计卡片展示。
- 页面内增加管理动作：创建、编辑、删除、状态切换（通过 Modal 表单完成）。
- 颜色字段在标签页通过 `Input(type="color") + HEX 文本输入` 双通道维护。
- 文案覆盖 `zh-CN/en-US/ja-JP` 三语。

### 2) 权限与数据隔离

- 分类/标签接口均要求登录（`Authorization: Bearer`）。
- 接口按 `ownerId` 隔离数据：只可读写当前登录用户拥有的数据。
- 列表接口与详情接口都不再返回“全局公共分类/标签”。

### 3) 菜单入口调整

- 用户菜单由“分类/标签浏览”改为“分类管理/标签管理”。
- 跳转路径改为：
  - `/{lang}/categories/manage`
  - `/{lang}/tags/manage`

## 关键实现约束

- 分类与标签创建时，`ownerId` 由服务端从当前登录用户写入，不接受前端透传。
- 更新与删除时，后端查询条件包含 `id + ownerId`，避免越权写操作。
- Hook 请求默认携带 `clientBearerHeaders()`，避免页面层遗漏鉴权头。
