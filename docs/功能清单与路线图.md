# 功能清单与路线图

本文档与**当前仓库实现**对齐；若与旧版手写说明冲突，以 `lib/db/schema.ts`、`app/api` 与页面代码为准。

## 已具备的能力（摘要）

### 认证与账户

- 用户名/邮箱登录、注册、邮箱验证码（JWT + `bcryptjs`）
- 忘记/重置密码链路已接入数据库 token、邮件发送、限流、密码强度校验与活动审计
- 密码字段支持 `passwordTransport`（AES-GCM + RSA-OAEP + HMAC + nonce）传输封装
- 角色：`super_admin` / `admin` / `author` / `user`（权限细节以各 API 校验为准）

### 内容与分类

- 文章 CRUD、状态（`draft` / `published` / `archived`）、可见性、浏览计数等
- 文章编辑：**Toast UI Editor**（见 `components/blog/simple-editor.tsx`），支持 Markdown 类编辑体验
- 分类（含 `parentId` 层级）、标签多对多、公开列表与管理页
- 列表「搜索」：主要为 **数据库 LIKE 等条件**，非 Elasticsearch 级全文检索

### 个人中心

- 资料、文章列表、**收藏**（`/api/profile/favorites`）、**点赞列表**（`/api/profile/likes`）、通知、活动等与 schema 对齐的功能
- 粉丝/关注列表、关注/取消关注、公开用户资料页与通知中心已接入真实 API

### 国际化与 UI

- 三种语言路由 + `getDictionary`；HeroUI + Tailwind 4；明暗主题

### 评论与审核

- 前台 `POST /api/comments` 支持匿名或登录提交；登录时作者身份以服务端 JWT 解析为准
- 评论默认 `pending`，命中基础反垃圾规则时进入 `spam`
- 超级管理员可通过 `/api/admin/comments` 和 `/api/admin/comments/{id}` 执行筛选、单条/批量审核、删除，并记录审核活动日志

### 媒体与对象存储

- `POST /api/uploads/image` / `DELETE /api/uploads/image` 支持登录用户上传/删除 MinIO/S3 兼容图片对象
- 文章创建/编辑页已接入特色图上传组件，scope 为 `article`；对象键按用户命名空间隔离
- `media` 表已定义，但当前图片上传链路主要直接写 `posts.featured_image` URL，尚未形成通用媒体库后台

### 开发者工具

- 内置 **`/{lang}/api-docs`**，扫描展示 `app/api` 路由（**仅超级管理员**可浏览与拉取元数据；支持 OpenAPI 导出，见 `docs/接口文档系统说明.md`）

### 数据库层已定义、产品化程度不一

以下能力在 `schema` 中已定义，但产品化深度不一致，需按具体需求评估：

- **评论嵌套展示**：`comments.parentId` 已定义，当前提交 API 未开放回复树完整产品体验
- **媒体**表 `media`：当前 MinIO 上传不写入通用媒体资产表，媒体库、复用、审计与清理策略仍待设计
- **系统设置**表（后台 UI 与读取路径视需求而定）

## 配置说明

- 根目录 **`env.example`** 仅保留当前已用键；**Redis 等**是否在运行时可查 **`deploy/env.docker.example`** 与 Compose，**是否被业务代码读取**以全局检索为准。

2. **中期**：媒体库与 `media` 表闭环；搜索升级；RSS/站点地图；管理仪表盘
3. **长期**：协作编辑、多租户、插件化（若仍需要）

## 技术债提示（非穷尽）

- 服务层与页面中仍有较多 `any`，建议先从 API 响应类型、`PostService` join 结果和 profile 组件逐步收紧
- 复杂查询建议配合索引与分页策略复查；文章列表标签加载优先做批量化
- 文档、字典与路由的新增语言需同步 `middleware.ts` 与 `generateStaticParams`
