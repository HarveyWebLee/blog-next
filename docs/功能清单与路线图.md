# 功能清单与路线图

本文档与**当前仓库实现**对齐；若与旧版手写说明冲突，以 `lib/db/schema.ts`、`app/api` 与页面代码为准。

## 已具备的能力（摘要）

### 认证与账户

- 用户名/邮箱登录、注册、邮箱验证码、忘记/重置密码（JWT + `bcryptjs`）
- 角色：`admin` / `author` / `user`（权限细节以各 API 校验为准）

### 内容与分类

- 文章 CRUD、状态（`draft` / `published` / `archived`）、可见性、浏览计数等
- 文章编辑：**Toast UI Editor**（见 `components/blog/simple-editor.tsx`），支持 Markdown 类编辑体验
- 分类（含 `parentId` 层级）、标签多对多、公开列表与管理页
- 列表「搜索」：主要为 **数据库 LIKE 等条件**，非 Elasticsearch 级全文检索

### 个人中心

- 资料、文章列表、**收藏**（`/api/profile/favorites`）、通知、活动等与 schema 对齐的功能

### 国际化与 UI

- 三种语言路由 + `getDictionary`；HeroUI + Tailwind 4；明暗主题

### 开发者工具

- 内置 **`/{lang}/api-docs`**，扫描展示 `app/api` 路由（**仅超级管理员**可浏览与拉取元数据；见 `docs/接口文档系统说明.md`）

### 数据库层已定义、产品化程度不一

以下在 **schema 中已存在**，HTTP/管理界面可能仍不完整，需在需求文档中单独划界：

- **评论**表 `comments`（暂无统一 `app/api/comments` 资源时可视为「待接 API」）
- **媒体**表 `media`（上传流水线、权限与前端媒体库视需求而定）
- **系统设置**表（后台 UI 与读取路径视需求而定）

## 配置说明

- 根目录 **`env.example`** 仅保留当前已用键；**Redis 等**是否在运行时可查 **`deploy/env.docker.example`** 与 Compose，**是否被业务代码读取**以全局检索为准。

## 待办与改进方向（按价值粗略分组）

### 内容与编辑

- [ ] 草稿**自动保存**（前端定时/失焦提交）与冲突处理
- [ ] **图片/附件上传**与 `media` 表、权限、CDN 策略闭环
- [ ] 文章版本历史、导入导出、批量操作（按需）

### 互动与审核

- [ ] **评论** REST API、嵌套展示、审核后台与垃圾过滤策略
- [ ] 点赞/阅读进度/分享等（`schema` 已有字段的可先做产品化）

### 发现与运营

- [ ] 更强搜索（全文索引、独立搜索服务或 MySQL FULLTEXT 等）
- [ ] RSS、站点地图、订阅邮件
- [ ] 管理端「仪表盘」聚合统计

### 性能与可靠

- [ ] 若启用 Redis：会话/热点列表缓存、限流（环境变量与 **`deploy/env.docker.example`** / 代码引用对齐）
- [ ] 图片压缩、懒加载策略统一
- [ ] 监控、审计日志、备份恢复演练

### 安全

- [ ] 视需要你方可补充 CSRF、更细粒度 RBAC、安全响应头等（以实际威胁模型为准）

## 分阶段建议（可随排期调整）

1. **近期**：评论 API + 前台列表；媒体上传最小闭环；自动保存
2. **中期**：搜索升级；RSS/站点地图；管理仪表盘
3. **长期**：协作编辑、多租户、插件化（若仍需要）

## 技术债提示（非穷尽）

- 编辑器 `simple-editor.tsx` 中部分 `any` 与 SSR 绕过可逐步收紧类型
- 复杂查询建议配合索引与分页策略复查
- 文档、字典与路由的新增语言需同步 `middleware.ts` 与 `generateStaticParams`
