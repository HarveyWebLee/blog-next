# 接口文档系统说明

## 概述

项目内置 **API 文档浏览站点**：自动扫描 `app/api` 下各 `route.ts`，在页面上分组展示方法、参数、请求体、鉴权说明与示例。数据由 `lib/utils/api-scanner.ts` 解析，`app/api/api-docs/route.ts` 对外提供 JSON，页面路由为 **`app/[lang]/api-docs`**。

补充元数据（端点中文说明、分组副标题、鉴权提示）集中在 **`lib/utils/api-docs-endpoint-meta.ts`**，由扫描器在生成 `ApiEndpoint` 时合并，避免 `api-scanner.ts` 无限膨胀。

## 访问控制（重要）

- **页面**：`/{lang}/api-docs` 使用 `useAuth()` 与 **`isInMemorySuperRootClientUser`**（`lib/utils/authz.ts`）判断：仅 **超级管理员 root 会话**（`role === "super_admin"`；使用 DB 真实 `user.id`）可查看内容与使用「测试接口」。
- **数据接口**：`GET /api/api-docs` 使用 **`requireInMemorySuperRoot`** 校验 `Authorization: Bearer`，与页面规则一致；**数据库管理员（admin）与普通用户不能拉取扫描结果**，避免泄露全站路由结构。
- 非授权用户访问页面时展示 **`ApiDocsAccessDenied`** 说明；勿在公开导航中链接本页。

## 功能概览

### 自动扫描

- 递归扫描 `app/api` 目录
- 识别导出函数对应的 HTTP 方法（GET、POST、PUT、DELETE、PATCH 等）
- 解析注释中的描述（建议在各 `route.ts` 中写清 JSDoc）
- 合并 `api-docs-endpoint-meta.ts` 中的端点说明与 **鉴权提示**（`authHint` 字段）
- 生成请求/响应示例与 cURL 提示

### 文档界面

- 响应式布局，支持明暗主题
- 多语言界面（`dictionaries/*` 中 `apiDocs` 相关键）
- 搜索、按 HTTP 方法筛选（搜索范围含鉴权说明文案）
- 分组折叠与接口详情展开

### 登录鉴权测试（仅超级管理员页内）

- **`ApiDocsAuthPanel`**：调用 `POST /api/auth/login`，将返回的 `token` 写入 **`ApiDocsTesterProvider`** 的上下文，并触发重新拉取 `/api/api-docs`（便于会话过期后仅在本页续测）。
- **「同步全站已登录 Token」**：从 `localStorage.accessToken` 读入同一上下文。
- **`ApiTester`**：打开测试模态时若尚未填写 `Authorization`，自动填入上下文中的 Bearer；亦可手动点击「填入文档测试 Token」。

  ...

lib/utils/api-docs-endpoint-meta.ts # 人工维护的说明与鉴权文案

## 使用方式

1. 在环境变量中启用并配置 **超级管理员**（见 `env.example` 中 `SUPER_ADMIN_*` 与 `lib/config/super-admin.ts`）。
2. 启动应用：`pnpm dev`
3. 使用该账号在 **`/{lang}/auth/login`** 登录后访问 **`/{lang}/api-docs`**。
4. 若仅在本页用「登录鉴权测试」登录，不必覆盖全站 `localStorage`；拉取文档数据会 **优先使用上下文 Token**，其次才是 `localStorage` 中的 `accessToken`。
5. 当扫描结果未更新时，可请求 **`/api/api-docs?refresh=true`** 强制重扫（接口本身仍要求超级管理员 Bearer）。

## 自定义与扩展

### 补充 API 分组 / 端点 / 鉴权说明

优先编辑 **`lib/utils/api-docs-endpoint-meta.ts`**：

- `API_DOCS_GROUP_DESCRIPTIONS`
- `API_DOCS_ENDPOINT_DESCRIPTIONS`
- `API_DOCS_AUTH_HINTS`

路径须与扫描器输出的 `apiPath` 一致（动态段形如 `{id}`、`{slug}`，`[...path]` 对应 `{...path}`）。

### 扫描器本体

在 `lib/utils/api-scanner.ts` 中仍可扩展参数提取、示例生成等逻辑。

### 新增界面语言

在 `dictionaries/` 增加语言文件，并补齐 `apiDocs` 相关翻译键。

## 注意事项

1. Route Handler 文件须命名为 **`route.ts`**
2. 复杂类型建议在 `types/` 中定义并在路由注释中引用说明
3. 路由数量很大时，服务端扫描结果带 **5 分钟内存缓存**（`?refresh=true` 强制重扫）

## 故障排除

| 现象                   | 排查                                                             |
| ---------------------- | ---------------------------------------------------------------- |
| 页面提示无权访问       | 确认当前登录用户为超级管理员（role=super_admin），非数据库 admin |
| 403 / 无法加载文档数据 | `GET /api/api-docs` 请求头是否带有效超级管理员 Bearer            |
| 扫描结果不完整         | 确认目录在 `app/api` 下且文件名为 `route.ts`                     |
| 界面文案缺失           | 检查对应语言的 `dictionaries/*.json`                             |
| 样式异常               | 确认 Tailwind 与主题 class 未冲突                                |
