# 接口文档系统说明

## 概述

项目内置 **API 文档浏览站点**：自动扫描 `app/api` 下各 `route.ts`，在页面上分组展示方法、参数、请求体与示例。数据由 `lib/utils/api-scanner.ts` 解析，`app/api/api-docs/route.ts` 对外提供 JSON，页面路由为 **`app/[lang]/api-docs`**。

> 具体暴露了哪些路径，以运行应用后访问 `http://localhost:3000/zh-CN/api-docs`（或 `en-US` / `ja-JP`）的扫描结果为准；本文不逐条罗列，以免与代码不同步。

## 功能概览

### 自动扫描

- 递归扫描 `app/api` 目录
- 识别导出函数对应的 HTTP 方法（GET、POST、PUT、DELETE、PATCH 等）
- 解析注释中的描述（建议在各 `route.ts` 中写清 JSDoc）
- 生成请求/响应示例与 cURL 提示

### 文档界面

- 响应式布局，支持明暗主题
- 多语言界面（`dictionaries/*` 中 `apiDocs` 相关键）
- 搜索、按 HTTP 方法筛选
- 分组折叠与接口详情展开

## 目录与关键文件

```
app/[lang]/api-docs/     # 文档页面（按语言）
components/api-docs/     # 文档 UI 组件
app/api/api-docs/        # 向扫描器请求元数据的 API
lib/utils/api-scanner.ts # 扫描与元数据生成
```

## 使用方式

1. 启动应用：`pnpm dev`
2. 浏览器打开：`/{lang}/api-docs`，例如 `/zh-CN/api-docs`
3. 导航栏中「API 文档」链接触达（若已配置）

## 自定义与扩展

### 补充 API 分组说明

在 `lib/utils/api-scanner.ts` 的 `getGroupDescription`（或等价映射）中为分组 id 增加描述文案。

### 的更贴切示例

在同一文件中按需扩展 `generateRequestBodyExample`、`generateResponseExample` 等，为特定路由生成更准确示例。

### 新增界面语言

在 `dictionaries/` 增加语言文件，并补齐 `apiDocs` 相关翻译键。

## 可与后续工作结合的方向

- 在文档页内直接「试调」接口（需处理鉴权与 CORS）
- 接口版本维度（v1/v2）
- 登录态/权限说明块
- 调用统计或变更记录

## 注意事项

1. Route Handler 文件须命名为 **`route.ts`**
2. 复杂类型建议在 `types/` 中定义并在路由注释中引用说明
3. 路由数量很大时，可考虑对扫描结果做缓存或分页（当前实现以开发友好为主）

## 故障排除

| 现象           | 排查                                         |
| -------------- | -------------------------------------------- |
| 扫描结果不完整 | 确认目录在 `app/api` 下且文件名为 `route.ts` |
| 界面文案缺失   | 检查对应语言的 `dictionaries/*.json`         |
| 样式异常       | 确认 Tailwind 与主题 class 未冲突            |
