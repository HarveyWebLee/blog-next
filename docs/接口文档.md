# 接口文档（全量清单）

本文档与仓库 **`app/api/**/route.ts`** 保持一致，用于快速检索路径、方法与鉴权要点。内置交互式文档站见 **`/{lang}/api-docs`**（扫描实现 + **`lib/utils/api-docs-endpoint-meta.ts`** 元数据）。**专项变更流水**见 **`docs/接口变更文档.md`\*\*。

**约定**：无前缀的 API 路径均指站点根路径下的 **`/api/...`**；鉴权以各 `route.ts` 实现为准，下表「鉴权要点」为摘要。

---

## 全局错误与日志约定（2026-04-23）

- `app/api/**/route.ts` 统一使用 `defineApiHandlers` 包装导出，集中处理访问日志、未捕获异常与响应兜底。
- 未捕获异常默认返回统一 JSON 500，基础结构为 `success=false`、`message`、`timestamp`，不再回落为 HTML 错误页。
- 各接口可通过 `onUnhandledErrorResponse` 覆盖默认 500 文案（按方法/场景返回业务友好提示），但不暴露底层 SQL 与堆栈细节。
- 统一日志走 `lib/server/logger.ts`，并结合 `lib/server/log-redaction.ts` 脱敏；数据库摘要日志通过 `logDbError` 输出，避免打印整段 Drizzle SQL。
- 可选告警桥接由 `lib/server/route-alert.ts` 提供（`LOG_ALERTS_ENABLED` / `LOG_ALERT_WEBHOOK_URL`）。

### profile 补充约定（2026-04-21）

- `PUT /api/profile` 修改邮箱时，需提交 `emailVerificationCode`（验证码由 `POST /api/auth/send-verification-code` 且 `type=change_email` 获取）。
- `socialLinks` 建议仅使用 `github`、`wechatQr`、`douyin`、`bilibili` 四个键；服务端读取兼容历史键名。
- 资料保存会同步更新 `users` 表中的 `email`、`avatar`、`displayName`，前端应以最新 `GET /api/profile` 返回为准刷新本地用户态。
