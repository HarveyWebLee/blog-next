## 2026-04-10 — Git 主分支使用 main

**类型**：Git / 工程约定

### 需求

将仓库默认主分支从 `master` 改为 **`main`**。

### 已做操作（本地）

1. **`git branch -m main`**：将当前分支名由 `master` 重命名为 `main`（本仓库当时尚无任何 commit 时亦可执行）。
2. **`git config init.defaultBranch main`**：本仓库本地配置中指定 **`git init` 时默认初始分支为 `main`**。

### 文档与规则

- **`.cursorrules`** 中 semantic-release 相关表述为仅写推送到 **`main`**。

### 远程仓库（若已存在）

若 GitLab/GitHub 上仍只有 `master`，首次 `git push` 后需在托管平台将**默认分支**设为 `main`，并按需删除或归档 `master`。

```bash
git push -u origin main
```

### 验证

- `git status -sb` 显示在 **`main`** 上。
