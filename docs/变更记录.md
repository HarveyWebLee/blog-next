## 2026-04-22 — Deploy 流水线增强：Git 工作区权限自愈与自动重建兜底

**类型**：fix（ci / deploy）

### 处理

- 更新 `.github/workflows/deploy.yml` 远端部署脚本，在 `git pull` 前新增“工作区健康检查与自愈”流程：
  - 先尝试当前用户执行 `chmod -R u+rwX`；

  - 若关键目录（`.git`、`app`）仍不可写，自动触发“工作区重建”：
    - 读取当前 `origin` 地址；
    - 临时克隆 `main` 到新目录；
    - 保留并迁移 `deploy/.env.docker`；
    - 原目录原子替换为新工作区。
- 目标是消除部署中 `git pull` 过程的 `unable to unlink old ... Permission denied` 类型阻断，避免必须手工登录服务器修复。

## 2026-04-10 — 修复 semantic-release 提交时 Husky 拦截 CHANGELOG

**类型**：CI / semantic-release

### 问题

### 处理

1. **`@semantic-release/exec`**：在 **changelog、npm** 插件之后，**git** 提交之前执行 **`prettier --write CHANGELOG.md package.json`**。
2. **`.github/workflows/release.yml`**：在 **semantic-release** 步骤设置 **`HUSKY=0`**，避免发布机器人提交再走本地钩子（与 Prettier 双保险）。
