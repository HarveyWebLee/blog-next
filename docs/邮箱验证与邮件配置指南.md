# 邮箱验证与邮件配置指南（按当前实现）

## 功能概述

当前仓库中的邮箱验证码能力覆盖以下场景：

1. **注册验证码**（`register`）
2. **重置密码验证码**（`reset_password`）
3. **个人中心改绑邮箱验证码**（`change_email`）
4. **访客订阅验证码**（`subscription`）
5. **访客取消订阅验证码**（`subscription_unsubscribe`）

## 技术实现

### 1. 验证码存储

验证码存储在 `email_verifications` 表，核心行为：

- 验证码 6 位数字；
- 默认 10 分钟有效；
- 同邮箱同类型 1 分钟内限发一次；
- 新码生成时会将旧的未使用验证码置为已使用；
- 验证成功后即置为已使用（一次性）。

### 2. API 端点

#### 发送验证码

- **端点**: `POST /api/auth/send-verification-code`
- **参数**: `{ email: string, type: string }`
- **功能**: 发送 6 位验证码到指定邮箱
- **支持类型**: `register` / `reset_password` / `change_email` / `subscription` / `subscription_unsubscribe`

#### 验证验证码

- **端点**: `POST /api/auth/verify-code`
- **参数**: `{ email: string, code: string, type: string }`
- **功能**: 校验并消费验证码

#### 注册（可选验证码）

- **端点**: `POST /api/auth/register`
- **参数**: `{ username, email, password, displayName, useEmailVerification, verificationCode }`
- **说明**: `useEmailVerification=true` 时需要 `verificationCode`

#### 资料改绑邮箱

- **端点**: `POST /api/profile`、`PUT /api/profile`
- **关键参数**: `email`、`emailVerificationCode`
- **说明**: 当邮箱发生变化时，后端会强制校验 `change_email` 类型验证码

#### 邮件订阅/退订（访客）

- **端点**: `POST /api/subscriptions`、`DELETE /api/subscriptions`
- **关键参数**: `email`、`verificationCode`
- **说明**:
  - 未登录访客必须先验证码校验后才能订阅/退订；
  - 已登录用户按账号邮箱直接订阅/退订（无需验证码）。

### 3. 邮件服务

项目使用 `nodemailer`（`lib/utils/email.ts`）发送邮件，支持多种 SMTP 服务商：

- Gmail
- Outlook
- 自定义 SMTP 服务器

## 配置说明

### 环境变量配置

在 `.env` 文件中添加以下配置：

```env
# 邮件配置
SMTP_HOST=smtp.gmail.com
SMTP_PORT=587
SMTP_SECURE=false
SMTP_USER=your_email@gmail.com
SMTP_PASS=your_app_password_here
```

### Gmail 配置示例

1. 启用两步验证
2. 生成应用专用密码
3. 使用应用专用密码作为 `SMTP_PASS`

### 其他邮件服务商配置

#### Outlook/Hotmail

```env
SMTP_HOST=smtp-mail.outlook.com
SMTP_PORT=587
SMTP_SECURE=false
```

#### QQ邮箱

```env
SMTP_HOST=smtp.qq.com
SMTP_PORT=587
SMTP_SECURE=false
```

## 使用流程（关键场景）

### 注册流程（可选验证码）

```
填写用户名/显示名/邮箱/密码
-> (可选) 发送 register 验证码并输入
-> POST /api/auth/register
```

### 个人中心改绑邮箱流程

```
输入新邮箱
-> POST /api/auth/send-verification-code(type=change_email)
-> 输入验证码
-> POST/PUT /api/profile 提交 email + emailVerificationCode
```

### 访客订阅流程

```
输入邮箱
-> POST /api/auth/send-verification-code(type=subscription)
-> 输入验证码
-> POST /api/subscriptions
```

### 验证码规则

- **长度**: 6位数字
- **有效期**: 10分钟
- **发送频率**: 1分钟内只能发送一次
- **使用次数**: 每个验证码只能使用一次

## 安全特性

### 1. 验证码安全

- 验证码随机生成，不可预测
- 设置有效期，防止重放攻击
- 限制发送频率，防止滥用
- 一次性使用，防止重复使用

### 2. 邮箱验证

- 验证邮箱格式
- 注册场景检查邮箱是否已被注册
- 验证码与邮箱绑定

### 3. 数据库安全

- 验证码明文存储于验证码表（由有效期、一次性消费与限频控制风险）
- 自动清理过期验证码
- 防止SQL注入

## 故障排除

### 常见问题

1. **邮件发送失败**
   - 检查 SMTP 配置
   - 确认邮箱密码正确
   - 检查网络连接

2. **验证码收不到**
   - 检查垃圾邮件文件夹
   - 确认邮箱地址正确
   - 检查邮件服务商限制

3. **验证码无效**
   - 检查验证码是否过期
   - 确认验证码输入正确
   - 检查是否已使用过
