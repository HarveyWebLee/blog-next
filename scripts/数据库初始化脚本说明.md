# 数据库初始化脚本说明（init-db.sql）

## 概述

`scripts/init-db.sql` 可在空库上执行，用于：

- 创建 `blog_system` 数据库与用户（脚本内定义的用户名/密码）
- 按脚本内容创建表并插入初始数据

## 与 Drizzle 的关系

本仓库**日常开发推荐**路径为：

1. 在 `lib/db/schema.ts` 维护结构
2. 使用 `pnpm db:generate` / `pnpm db:migrate` 或 `pnpm db:push`

若 **同时使用** `init-db.sql` 与 Drizzle 迁移，可能出现重复建表或结构不一致。**请团队内约定只保留一条主线**，或确保 SQL 脚本仅作一次性引导、后续以 Drizzle 为准。

## 使用方法

### Docker 内 MySQL

```bash
docker ps   # 确认容器名，例如 mysql-server 或 blog-mysql
docker exec -i <容器名> mysql -u root -p < scripts/init-db.sql
```

### 本机 mysql 客户端

```bash
mysql -u root -p < scripts/init-db.sql
```

## 验证

```bash
pnpm test:db:connect
```

查看表：

```bash
docker exec -it <容器名> mysql -u root -p -e "USE blog_system; SHOW TABLES;"
```

## 安全提示

- 脚本中的默认密码**仅用于开发**，生产环境必须修改
- 执行前若库内已有数据，请先备份（可用 `pnpm db:backup`）
