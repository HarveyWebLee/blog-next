# Security Env Boundary Check

## 适用场景

- 变更鉴权逻辑（JWT、密码、会话）或 API 权限控制。
- 新增或修改环境变量读取代码。
- 邮件发送、数据库连接、密钥管理相关改动。

## 不处理范围

- 不负责发布流程检查（交给 `release-readiness-check`）。
- 不负责提交信息生成（交给 `commit-message-angular-cn`）。
- 不负责 i18n 一致性（交给 `i18n-consistency-guard`）。

## 执行步骤

1. 检查敏感信息边界
   - 密钥、密码、令牌不得写入前端可见代码或响应体。
2. 校验环境变量分层
   - 仅公开配置使用 `NEXT_PUBLIC_*`；其余仅服务端读取。
3. 校验鉴权与错误返回
   - 确认接口对未授权场景返回一致且不泄露内部细节。
4. 检查日志输出
   - 不输出明文凭证、token、邮件密钥等敏感内容。

## 完成标准

- 无明显敏感数据泄露路径。
- 服务端与客户端环境变量边界清晰。

## 最小使用示例

- 示例触发语句：`帮我检查这次 JWT 登录改动有没有把敏感信息暴露到前端`
- 预期行为：输出安全边界检查结论，不代替提交信息或发版判断。
