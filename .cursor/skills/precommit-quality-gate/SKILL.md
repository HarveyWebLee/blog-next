# Precommit Quality Gate

## 适用场景

- 功能改动完成后的收尾阶段。
- 用户要求“检查一下能不能提交”。

## 不处理范围

- 不生成 commit message（交给 `commit-message-angular-cn`）。
- 不做接口和数据库文档同步决策（分别交给 `api-contract-sync`、`db-schema-sync`）。
- 不做发版语义分析（交给 `release-readiness-check`）。

## 执行步骤

1. 选择最小检查集合
   - 至少包含 `pnpm lint` 与必要的格式检查。
2. 运行并记录结果
   - 提供通过/失败结论与关键错误摘要。
3. 失败时给出修复优先级
   - 先修阻断项，再处理告警项。
4. 输出提交门禁结论
   - `可提交` / `不可提交`，并附最少行动建议。

## 完成标准

- 结论明确，且与命令结果一致。
- 不在此 skill 内处理提交内容撰写与发版策略判断。

## 最小使用示例

- 示例触发语句：`改完了，帮我跑一下提交前检查，告诉我现在能不能提`
- 预期行为：执行质量检查并输出可提交/不可提交结论。
